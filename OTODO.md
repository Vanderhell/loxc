## 2026-05-12 — Fáza 8: KROK 3 & KROK 4 — Frequency Analysis & Header Generation

### Status: ✅ KROK 3 COMPLETED — Strategy Selection, ✅ KROK 4 COMPLETED — Header File Generation

#### KROK 3: Frequency Analysis (analyze_freqs)

Implemented complete frequency analysis pipeline:
1. **Uni-gram counting**: Byte-level character frequency analysis (not UTF-8)
2. **Dictionary analysis**: Called `loxc_dict_analyze()` to find word patterns
3. **Character subtraction**: Dict entries are removed from char counts to avoid double-counting
4. **Symbol array building**: 
   - Char symbols (if count > 0)
   - Dict symbols (if ref_id != 0xFFFFu, i.e., gain > 0)
5. **Symbol sorting**: By count descending (frequency-ranked)

**Results for trainings/english.txt (738,046 bytes):**
- Total chars in text: 738,046
- Chars covered by dict: ~568,000 (77%)
- Unique symbols: 4,105 (34 chars + 4,071 dict entries)
- Dict entries analyzed: 7,471 total, 4,071 accepted (gain > 0), 3,400 rejected
- Overhead: 95,213 bytes (12.9% of input)

**Key discovery — Dict Threshold Optimization:**
- Entries with count=1 have negative gain (cost to include > benefit)
- Solution: Only accept entries where ref_id != 0xFFFFu (marked by loxc_dict_analyze)
- Result: Reduced overhead from 31.7% (accepting all) to 12.9% (accepting gain > 0)
- **Future optimization**: Consider power-of-2 boundary (reduce 4,105 to 4,096 for 13→12 bits/symbol)

#### KROK 3: Strategy Selection

Called `loxc_strategy_select()` to pick optimal encoding strategy:
- **Selected strategy**: FLAT_FIXED_WIDTH
- **Predicted bits**: 3,489,486 bits (436 KB encoded data)
- **Bits per symbol**: 13.01 (ceil(log2(4105)))
- **Total output**: 531,399 bytes (72.0% of input = 28% savings)

**Why FLAT won over HIER8:**
- HIER8 would require 74 levels (to encode 4,105 symbols)
- Escape overhead at L0-L73: 74 × 6 bits = 444 bits (unproductive)
- FLAT: 13 bits/symbol × all symbols (no escape overhead)
- Mathematical minimum wins regardless of distribution shape
